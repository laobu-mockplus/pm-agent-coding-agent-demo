# 小五工作台 UI 设计

## 目标

本次设计目标是把“小五 PM Agent 管理 CC Coding Agent 完成 SmallCalc MVP”的过程从命令行和 GitHub 页面，转成一个可视化、可重放的 UI 工作台。用户可以在界面中一步步推进流程，看到每一步的角色、状态、产物、证据和结果。

## 范围

本阶段实现的是稳定可重放的 demo UI，不直接触发真实 GitHub 或真实 Codex 运行。界面中的步骤、状态、链接描述和验收结果基于当前已经跑通的 SmallCalc demo 流程整理，目的是验证产品表达和交互模型。

## 7 步流程

1. 小五创建 SmallCalc PRD。
2. 小五给 CC 安排 MVP 任务。
3. CC 完成第一次实现并提交报告。
4. 小五第一次验收。
5. 小五判定不通过并列出不合格项。
6. CC 根据验收结果修复并再次提交。
7. 小五再次验收并通过。

## UI 结构

- 顶部栏：展示小五工作台名称、流程控制按钮。
- 状态条：展示当前步骤、当前角色、流程进度和验收状态。
- 左侧流程区：展示 7 步时间线，支持点击跳转。
- 中央详情区：展示当前步骤的说明、产物、证据和结果。
- 右侧日志区：展示已经执行过的步骤结果，形成可追踪历史。

## 布局约束

该界面属于生产工具型 UI，采用桌面式布局。外层应用壳固定为浏览器可视窗口高度，主工作区内部滚动，不依赖 `body` 或 `main` 的页面级滚动承载内容。

## 后续替换方向

当前步骤数据是 demo 固定数据。后续正式版本应把步骤状态接入真实编排运行记录，包括 GitHub Issue、PR、review、Codex/Qoder 执行日志、测试结果和小五验收结论。
